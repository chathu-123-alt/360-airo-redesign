# 360 AIRO - Frontend Engineering Assignment

## 🚀 Overview
This repository contains a full frontend UI redesign of the 360 AIRO authentication flow and an exploratory dashboard concept. My goal was to demonstrate modern UI/UX principles, component design, and premium aesthetic execution for a B2B SaaS platform.

## 🛠 Tech Stack
- **HTML5** (Semantic structure)
- **Pure CSS** (Custom variables, Flexbox/Grid, CSS micro-animations, Glassmorphism)
- **Vanilla JavaScript** (Basic interactivity)

## 🧠 UX/UI Analysis & Redesign Choices
During the research phase for this assignment, I encountered the "Account Approval Pending" state, which blocked access to the main dashboard. To demonstrate my frontend skills effectively, I made the following strategic choices:

1. **Authentication Flow Redesign (`index.html`, `signup.html`, `approval.html`)**: 
   - I completely overhauled the login, signup, and approval screens. 
   - Instead of a traditional split layout, I opted for a highly premium, dark-mode aesthetic with glassmorphism to align with the expectations of modern B2B developer and sales tools.
   - Added subtle CSS-driven micro-animations (glowing backgrounds, button hover scales) to make the interface feel alive and polished.

2. **Hypothetical Dashboard Creation (`dashboard.html`)**: 
   - To showcase product thinking without having dashboard access, I designed and coded a hypothetical "Email & LinkedIn Automation" dashboard.
   - Features a clean sidebar navigation and core metrics (Reply Rates, Active Campaigns) that users would expect in this product category.
   - Included a scalable CSS grid layout and a mock visual chart.

## 💻 How to Run
Because this project emphasizes pure UI/UX frontend skills without heavy frameworks, running it is incredibly simple:

1. Clone this repository to your local machine.
2. Double-click `index.html` to open it in any modern web browser.
3. No build steps, `npm install`, or local servers are required.
